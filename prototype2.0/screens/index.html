Szallas.hu UX prototype

Dashboard wszystkich ekranów (Plan: 21 kroków · zrealizowane: 21/21). Każdy ekran otwiera się w pełnym widoku.

User flow: rozpocznij od Strony głównej → wpisz „Balaton" w wyszukiwarce → przejdź do Listingu → kliknij hotel → otwórz Stronę hotelu (PDP) → wybierz pokój → przejdź do Koszyka → Płatności → Potwierdzenia.
🎚 Tweaks synchronizacja: Otwórz PDP i panel Tweaks (prawy-dolny róg). Zmiany tweaków (priceMode, loggedIn=VIP, urgency, showVipTeaser) są zapisywane do localStorage i widoczne na wszystkich pozostałych ekranach (listing/homepage/koszyk/płatność/thank you) jako odpowiednie dostosowania cen, wyświetlanych elementów + pill „PDP tweaks aktív" w lewym-dolnym rogu każdej strony.
📦 Master library: Strona hotelu (PDP) jest source-of-truth dla design tokenów (kolory, font, radii, shadows) oraz komponentów wspólnych (brandbar z „Úgy is lett!" tag, search bar, footer, score circle, meal pill, stepper). Pozostałe 5 ekranów importuje te same tokeny i wzorce HTML. W produkcyjnej aplikacji React te komponenty zostaną wyciągnięte jako shared modules (BrandBar, SearchBar, Footer, HotelCard, Stepper, MealPill itd.).
Strona główna → Listing → Strona hotelu (PDP) → Koszyk → Płatność → Potwierdzenie

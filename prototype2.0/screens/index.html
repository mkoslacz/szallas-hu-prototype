Szallas.hu UX prototype

Dashboard for all screens (Plan: 21 steps · completed: 21/21). Each screen opens in full view.

Note: The prototype UI is in Hungarian (the product targets Hungary). This dashboard and the Tweaks panel are in English for stakeholder review. If your browser auto-translates the Hungarian screens, disable translation to see the intended copy.
User flow: start at the Homepage → type "Balaton" in the search bar → continue to the Listing → click a hotel → open the Hotel page (PDP) → pick a room → continue to Checkout → Payment → Confirmation.
🎚 Tweaks sync: Open the PDP and the Tweaks panel (bottom-right). Tweak changes (priceMode, loggedIn=VIP, urgency, showVipTeaser) are stored in localStorage and reflected on every other screen (listing / homepage / checkout / payment / thank-you) as the corresponding price & element adjustments, plus a "PDP tweaks aktív" pill in the bottom-left of every page.
📦 Master library: The hotel page (PDP) is the source of truth for design tokens (colors, font, radii, shadows) and shared components (brandbar with the "Úgy is lett!" tag, search bar, footer, score circle, meal pill, stepper). The other 5 screens import the same tokens and HTML patterns. In a production React app these will be extracted as shared modules (BrandBar, SearchBar, Footer, HotelCard, Stepper, MealPill, etc.).
